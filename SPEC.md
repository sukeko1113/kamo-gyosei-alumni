# 機能仕様書（SPEC）

加茂暁星高等学校 同窓会ウェブサイトの機能仕様。実装は 3 つのフェーズに分けて進める。
本ドキュメントは実装の進行に合わせて随時更新する。

## 全体の方針

- **対象利用者**: 卒業生。高齢の方が多い。
- **アクセシビリティ最優先**: 文字は最小 16px、コントラストを確保し、操作を単純に保つ。
- **セキュリティ**: 秘密情報は環境変数で管理。microCMS の API キーはサーバー専用。

---

## フェーズ1：会員基盤

最初に「誰が会員か」を扱える状態にする。

### 1-1. Google アカウントでのログイン
- Firebase Authentication の Google プロバイダを用いたログイン／ログアウト。
- ログイン状態をアプリ全体で参照できるようにする（認証コンテキスト）。
- 未ログインでも閲覧できるページと、ログインが必要なページを区別する。

### 1-2. 会員プロフィール
- 初回ログイン時に Firestore へ会員ドキュメントを作成する。
- プロフィール項目（暫定）: 表示名、メールアドレス、卒業年度。
  - 型は `types/index.ts` の `User` を参照。
- 本人がプロフィールを閲覧・編集できる。
- Firestore セキュリティルールで「本人のみ自分のデータを読み書き可」を担保する。

---

## フェーズ2：情報発信と寄付

会員基盤の上に、情報発信と資金集めの仕組みを載せる。

### 2-1. お知らせ・イベント記事の表示（microCMS）
- microCMS の `blogs` エンドポイントから記事を取得して表示する。
  - 取得は `lib/microcms.ts`（サーバー専用）の `getBlogs` / `getBlogDetail` を使用。
- 一覧ページ（新着順）と詳細ページを用意する。
- カテゴリ（お知らせ／イベント等）で絞り込めるようにする。
- 記事の本文はリッチエディタの HTML を安全に表示する。

### 2-2. 寄付集め（Stripe）
- Stripe を用いた寄付（ワンタイム決済）の受付。
- 金額の選択、決済画面への遷移、完了・キャンセルの結果表示。
- 決済処理はサーバーサイド（Route Handler）で行い、秘密鍵は環境変数で管理する。
- ※ Stripe 関連の環境変数はこのフェーズ着手時に `.env.local.example` へ追記する。

---

## フェーズ3：参加と名簿

同窓会活動への参加と、卒業生どうしのつながりを支援する。

### 3-1. 耐震問題の署名機能
- 校舎の耐震問題に関する署名をオンラインで集める。
- ログイン会員が署名でき、署名内容（賛同の意思・任意コメント等）を Firestore に保存。
- 現在の署名数を表示する。重複署名を防ぐ。

### 3-2. 卒業生名簿
- 会員が同窓生を検索・閲覧できる名簿。
- 公開範囲は本人が選べるようにする（プライバシー配慮）。
- Firestore セキュリティルールで、ログイン会員のみ閲覧可とする。
